Never attempt to build large features in a single step.

Always:

1. Break features into small, testable tasks.
2. Explain the implementation plan before coding.
3. Build one component or feature at a time.
4. Verify that the current implementation works before moving to the next step.
5. Prefer many small successful changes over one large complex change.
6. When a feature is large, create a checklist and complete it step by step.
7. Avoid modifying unrelated files.
8. Keep changes focused and easy to review.
9. At the end of each task, explain what was completed and what should be built next.

Think like a senior engineer building a production application incrementally.
