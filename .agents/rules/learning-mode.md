Before creating any new folder, file, component, hook, utility, service, or configuration file:

1. Explain WHY it is needed.
2. Explain what responsibility it has.
3. Explain why existing files cannot be reused.
4. Show where it fits in the project architecture.
5. Wait for approval before creating large architectural changes.

When generating code:

- Explain the purpose of each file.
- Explain the folder structure.
- Explain component relationships.
- Explain data flow.
- Prefer simple solutions over complex abstractions.
- Do not create folders or files unless there is a clear reason.

Assume I am learning software engineering and want to understand every architectural decision.
